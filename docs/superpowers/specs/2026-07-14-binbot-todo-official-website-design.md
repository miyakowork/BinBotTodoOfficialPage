# BinBot Todo 官方网站设计规格

## 目标

为 BinBot Todo 建立一个面向中文用户的 macOS 产品官网。首版首先让访客理解它是一款简洁优雅、本地优先的待办事项工具，再通过真实产品工作流说明标签、优先级、截止时间、附件、搜索、AI 辅助与数据导出等能力。

BinBot Todo 当前没有公开 GitHub Release，因此首版不提供下载按钮、不承诺发布日期，也不收集邮箱。页面使用“持续开发中”和“关注项目进展”描述当前状态，并链接到 GitHub 项目。

## 产品依据

官网内容以相邻的 `BinBotTodo` 产品仓库为来源。当前产品版本为 2.1.0，使用 Tauri、React 和 TypeScript 构建，面向 macOS 10.15 及以上版本。

已实现并可用于官网介绍的能力包括：

- 待办创建、编辑、完成、删除与状态管理。
- 按今天、明天、本周、之后、已过期和已完成分组。
- 自定义标签、颜色、优先级和截止时间。
- 即时搜索、状态过滤与标签过滤。
- 备注、链接和图片附件。
- 跟随系统、浅色和深色主题。
- 全局快捷键与 macOS 原生窗口体验。
- JSON 和 Markdown 数据导出。
- 可选 AI 功能：根据描述或图片拆解待办，根据任务上下文生成标题。
- DeepSeek 与通义千问 Qwen 服务商配置；API Key 由用户自行提供。

## 受众与成功标准

核心受众是希望用轻量桌面工具管理工作与生活、重视界面品质和本地数据掌控的 macOS 用户。

首版成功标准：

- 用户在首屏内理解 BinBot Todo 是一款 macOS 待办工具，而不是在线服务或纯 AI 产品。
- 用户能快速识别日常管理、任务详情、AI 辅助和本地数据四类价值。
- 页面只介绍产品仓库中已经存在的能力，不虚构云同步、协作、移动端或下载状态。
- GitHub 是唯一外部转化入口，所有相关入口行为一致。
- 页面在桌面、平板和手机上完整可用，并尊重减少动态效果的系统设置。
- 项目可以静态构建和托管，不依赖后端、远程图片或运行时第三方服务。

## 核心定位与文案方向

首屏核心定位是“简洁优雅的 macOS 待办工具”，AI 作为后续增强能力，不作为产品的第一身份。

建议首屏主标题为“把今天，安排得刚刚好。”，辅助文案说明 BinBot Todo 将任务、标签、截止时间和细节整理进一个清晰、安静的 Mac 工作台。主行动入口为“看看它如何工作”，滚动到产品工作流；次行动入口为“关注项目进展”，打开 GitHub 项目。

页面整体叙事顺序为：先让用户感到轻松，再展示日常效率，然后介绍 AI 加成，最后建立本地数据与用户掌控的信任。

## 视觉方向

采用已确认的 A 方案“清澈原生”。页面以冰白、雾蓝和 BinBot Todo 图标中的清亮蓝为主，少量暖橙用于完成和优先级反馈。视觉气质明亮、安静、清楚，接近 macOS 原生产品表达，但通过大幅产品窗口、非对称构图和精细动效保持品牌记忆点。

主品牌蓝以 `#1877F2` 为基准，并根据对比度生成深色文字蓝、浅色背景蓝和焦点环色。页面不使用大面积紫色渐变，不采用通用卡片矩阵作为主要叙事结构。

字体使用本地字体栈，以 `Avenir Next`、`PingFang SC` 和兼容回退字体为主，不依赖远程字体。标题使用紧凑字距和明显层级，正文保持舒适行宽与充足留白。

背景使用低对比点阵、柔和蓝色光晕和极淡纹理。装饰效果不得影响文字对比度，移动端降低纹理与光晕强度。

## 信息架构

页面采用中文单页结构：

1. 固定透明导航：品牌、功能、AI、本地数据、GitHub。
2. 首屏：主标题“把今天，安排得刚刚好。”、产品定位、章节入口、GitHub 入口和大型应用窗口。
3. 日常工作流：创建、分类、优先级、截止时间、搜索和状态过滤。
4. 任务详情：备注、标签、链接和图片附件集中在单项任务中。
5. AI 辅助：从描述或图片拆解待办，根据上下文生成标题；明确功能可选且由用户配置服务商。
6. 本地与原生：Tauri 桌面体验、本地数据、系统主题和全局快捷键。
7. 数据掌控：JSON 和 Markdown 导出。
8. 项目状态：持续开发中，并引导用户关注 GitHub 项目。
9. FAQ：平台、系统要求、AI、数据位置、开源仓库和当前发布状态。
10. 页脚：品牌、项目状态、GitHub 与必要说明。

## 关键视觉组件

首屏右侧使用响应式产品演示，而不是一张在手机上无法阅读的固定桌面截图。桌面尺寸保留 BinBot Todo 的侧栏、时间分组列表和任务详情三栏结构，并使用真实产品视觉变量和控件形态。手机尺寸将演示重组为任务列表与详情卡片，保持文字可读。

后续章节使用“产品界面片段加说明文字”的交替布局：

- 工作流章节展示时间分组、完成框、优先级和标签。
- 任务详情章节展示标题、截止时间、备注、链接和附件。
- AI 章节展示描述输入、生成结果和可选服务商状态。
- 本地数据章节用文件与导出界面表达用户掌控，不暗示云同步。

应用图标直接复用 BinBot Todo 的正式资源。产品演示由本地组件绘制，避免远程图片依赖，也便于响应式与动效控制。

## 交互与动态效果

- 导航链接平滑滚动到对应章节，并提供清晰的键盘焦点样式。
- 移动端导航使用可通过键盘操作的折叠菜单，并维护正确的 `aria-expanded` 状态。
- 首屏按导航、标题、说明、行动入口和产品窗口依次进入。
- 章节进入视口时轻微上浮并显现；动效只执行一次，避免滚动干扰。
- 产品演示中的完成框执行一次短促的勾选动画，用于强调完成反馈。
- GitHub 入口在新标签页打开，并包含安全的 `rel` 属性。
- 当系统启用 `prefers-reduced-motion` 时，禁用非必要入场、平滑滚动和装饰性位移。

## 技术架构

使用 React 18、TypeScript 和 Vite 构建纯静态单页。首版不使用后端、数据库、账号系统、邮件服务或分析统计。

建议组件边界：

- `Navigation`：桌面与移动导航。
- `Hero`：首屏定位、行动入口和产品窗口。
- `ProductWindow`：可响应式的产品界面演示。
- `WorkflowSection`：日常任务管理能力。
- `TaskDetailSection`：任务内容与附件能力。
- `AISection`：可选 AI 工作流。
- `LocalFirstSection`：本地、原生与主题能力。
- `DataControlSection`：导出与数据掌控。
- `ProjectStatus`：当前项目状态和 GitHub 入口。
- `Faq`：高频问题。
- `Footer`：品牌与必要链接。

文案、导航、FAQ、项目状态和 GitHub 地址集中在本地内容配置中。交互状态仅包括移动导航、FAQ 展开项和一次性视口显现状态，均保存在浏览器内存中，不持久化。

动画优先使用 CSS 和轻量 Intersection Observer，不引入不必要的动画运行时。应用演示使用语义化 HTML 与 CSS 绘制，不连接真实产品数据库，也不模拟可造成误解的完整应用操作。

## 内容边界

首版仅使用中文并介绍 macOS 产品，不包含：

- 真实下载或安装包地址。
- 发布日期、倒计时或版本承诺。
- Windows、Linux、iOS 或 Android 支持。
- 云同步、多人协作、账号、订阅或支付。
- 邮件收集、分析统计、广告或远程内容源。
- 在线可编辑的完整 Todo 应用。

未来提供 GitHub Release 后，可以通过统一项目状态配置增加下载入口，而不改变页面主体结构。

## 错误与降级

- 静态正文和章节不依赖网络；JavaScript 未执行时主要内容仍可阅读。
- GitHub 不可访问时只影响外部跳转，不影响页面内容。
- 装饰图形或产品图标加载失败时不得阻断正文和导航。
- 未知发布信息统一显示“持续开发中”，不展示猜测日期。
- Intersection Observer 不可用时内容默认可见。

## 响应式与无障碍

重点验证约 1440px 桌面、768px 平板和 390px 手机视口。页面使用语义化标题层级、导航、章节、按钮和页脚；保证文本对比度、键盘可操作性、可见焦点和合理触控目标。

产品演示中的纯装饰图形对辅助技术隐藏，关键能力由附近正文表达。FAQ 使用原生或等价语义的可访问折叠结构。移动导航打开时保持合理焦点顺序，并支持 Escape 关闭。

## 验证策略

- TypeScript 类型检查和 Vite 生产构建。
- DOM 或组件测试覆盖移动导航、FAQ、GitHub 入口、关键文案和项目状态。
- 检查 1440px、768px 和 390px 视口下的布局、溢出、文字可读性与视觉层级。
- 检查键盘导航、可见焦点、语义结构和 `prefers-reduced-motion`。
- 确认生产构建不依赖远程图片、后端服务或分析脚本。
- 使用 Lighthouse 或等价检查关注可访问性、最佳实践和基础性能问题。

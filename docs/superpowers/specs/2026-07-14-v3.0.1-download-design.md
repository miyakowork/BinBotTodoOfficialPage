# BinBot Todo v3.0.1 官网下载设计

## 目标

将已经公开发布的 BinBot Todo v3.0.1 Apple Silicon DMG 接入官方网站，让用户可以从首页直接下载安装包，同时能查看 GitHub Release 发布说明，并准确了解架构与签名状态。

## 发布元数据

官网内容层统一维护以下发布信息：

- 版本：`3.0.1`
- 平台：macOS 10.15+
- 架构：Apple Silicon（arm64）
- DMG：`BinBot.Todo_3.0.1_aarch64.dmg`
- 下载地址：GitHub Release 的固定资产 URL
- 发布说明地址：GitHub Release `v3.0.1` 页面
- 签名状态：未使用 Apple Developer ID 签名，未经过 Apple 公证

组件从统一发布元数据读取 URL 和文案，不在多个组件中重复硬编码版本地址。

## 首页入口

### Hero

- 主操作按钮改为“下载 macOS 版”，直接下载 v3.0.1 ARM64 DMG。
- 主按钮明确标注 Apple Silicon，避免 Intel Mac 用户误下载。
- 次级操作为“查看发布说明”，在新标签页打开 GitHub Release。
- 产品特点区域保持现有视觉结构。

### 发布状态区

- 将“持续开发中”改为“v3.0.1 已发布”。
- 文案说明当前提供 Apple Silicon 版本。
- 提供直接下载和查看 GitHub Release 两个入口，与 Hero 保持一致。

## FAQ 与风险说明

- “现在可以下载吗？”更新为可以从官网直接下载 v3.0.1。
- 明确当前安装包仅适用于 Apple Silicon（arm64）。
- 新增或补充 Gatekeeper 说明：安装包未签名、未公证，首次打开可能显示 macOS 安全提示。
- 不提供绕过安全机制的自动脚本或误导性“已验证”表述。

## 可访问性与下载行为

- 下载链接使用普通 `<a>`，保持键盘可达性和浏览器原生下载行为。
- 外部 Release 链接使用 `target="_blank"` 和 `rel="noreferrer"`。
- 下载按钮的可访问名称包含平台或架构信息。
- 不在网站仓库复制 DMG，资产继续由 GitHub Release 托管，避免 GitHub Pages 仓库膨胀和重复版本管理。

## 测试与部署

- 内容测试验证发布状态、版本、架构及两个 URL。
- 页面测试验证 Hero 和状态区存在直接下载入口与 Release 入口。
- 页面测试验证旧的“暂无公开下载版本”文案不再出现。
- 运行 Vitest 全量测试与 Vite 生产构建。
- 推送官网 `main` 后等待 GitHub Pages 工作流成功。
- 在线验证首页状态码、下载链接地址和 Release 链接地址。

## 非目标

- 不创建独立下载页面。
- 不新增 Intel x64 或 Universal 安装包。
- 不重新打包、签名或公证 v3.0.1 DMG。
- 不改变官网整体视觉系统或页面信息架构。
